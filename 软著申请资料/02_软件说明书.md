# AI小说助手软件V1.0 操作手册

---

## 1 引言

### 1.1 编写目的

本操作手册面向软件使用者和软件著作权登记审查人员，详细说明AI小说助手软件的功能模块、操作流程和使用方法，帮助用户快速掌握软件的使用。

### 1.2 软件概述

AI小说助手是一款面向网络文学创作者的AI辅助写作桌面应用程序。软件集成大语言模型API能力，提供从项目创建、大纲编写、章节生成到人物管理的全流程写作辅助功能，帮助创作者高效完成长篇网络小说的创作。

### 1.3 运行环境

- **操作系统**：Windows 10及以上版本
- **硬件要求**：4GB以上内存，500MB以上可用磁盘空间
- **网络要求**：需连接互联网以访问AI大语言模型API服务

---

## 2 软件安装与启动

### 2.1 安装

双击安装包文件，按照安装向导提示完成安装。安装完成后，桌面和开始菜单将生成"AI小说助手"快捷方式。

### 2.2 首次配置

首次启动软件后，需在"设置"页面配置AI服务连接信息：

1. 点击左侧导航栏底部的"设置"按钮进入设置页面。
2. 在"AI Provider"区域，点击"添加Provider"按钮。
3. 填写Provider名称（如"OpenAI"、"DeepSeek"等）、API Base URL、API Key、模型名称。
4. 可选配置参数：Temperature（生成随机性，0-2）、Max Tokens（最大输出长度）、Top P（核采样参数）、上下文长度。
5. 点击"测试连接"按钮验证配置是否正确。
6. 测试成功后，该Provider自动激活，即可开始使用AI功能。

## 3 功能模块操作说明

### 3.1 项目管理

#### 3.1.1 创建项目

**操作路径**：首页 → 点击"新建项目"按钮

**操作步骤**：

1. 在弹出的创建项目对话框中，填写以下信息：
   - **项目名称**：输入小说名称（必填）
   - **小说类型**：从下拉列表中选择，支持玄幻、奇幻、武侠、仙侠、都市、历史、军事、游戏、科幻、悬疑、言情、二次元、其他共13种类型
   - **写作风格**：选择风格，支持轻松、严肃、热血、黑暗、幽默、文艺、史诗共7种
   - **目标字数**：设置小说总目标字数（万字）
   - **目标平台**：选择发布平台，支持番茄小说、起点中文网、晋江文学城、纵横中文网、七猫小说、其他共6种
   - **叙事视角**：选择第一人称或第三人称
   - **受众群体**：选择男频或女频
   - **故事节奏**：选择慢热、中速、快节奏或极速共4种
   - **项目描述**：简要描述小说核心设定和故事梗概
2. 点击"创建"按钮，系统自动在本地创建项目目录结构。

**AI辅助创建**：

在创建项目对话框中，可点击"AI辅助创建"按钮，输入简要的故事创意描述，AI将自动生成完整的世界观设定、总纲、支线剧情和初始人物设定，大幅缩短项目初始化时间。

#### 3.1.2 打开项目

在首页项目列表中，点击项目卡片即可进入项目详情页面。

#### 3.1.3 删除项目

在首页项目列表中，将鼠标悬停在项目卡片上，点击右上角的删除图标，确认后删除项目及其所有本地数据。

---

### 3.2 大纲管理

**操作路径**：项目详情 → 左侧导航 → 大纲

大纲管理支持三种类型的文档：

#### 3.2.1 总纲

总纲是小说的全局故事框架，包含主线剧情、关键转折点、结局走向等核心内容。

**操作步骤**：

1. 在大纲页面点击"总纲"选项卡。
2. 在Markdown编辑器中输入或编辑总纲内容。
3. 编辑器支持Markdown格式工具栏（加粗、斜体、引用、分割线）。
4. 编辑完成后，系统在2秒无操作后自动保存，也可按Ctrl+S手动保存。

#### 3.2.2 世界观

世界观文档记录小说的世界设定，包括地理、历史、力量体系、社会制度等背景信息。

操作方式同总纲。

#### 3.2.3 支线

支线文档记录小说的副线剧情、角色个人故事线等内容。

操作方式同总纲。

#### 3.2.4 AI润色大纲

**操作步骤**：

1. 在任意大纲编辑页面，点击"AI润色"按钮。
2. AI将深度分析当前大纲内容，从故事结构、人物动机、剧情冲突、节奏把控等维度进行优化。
3. 润色结果以流式输出方式实时显示在预览区域。
4. 查看润色结果后，可选择"应用"将结果替换当前大纲，或选择"丢弃"保留原内容。

#### 3.2.5 AI扩写大纲

**操作步骤**：

1. 在任意大纲编辑页面，点击"AI扩写"按钮。
2. AI将把简要大纲扩展为详细大纲，补充场景描写、对话要点、情绪变化等细节。
3. 实时预览扩写结果，可选择"应用"或"丢弃"。

---

### 3.3 卷管理

**操作路径**：项目详情 → 左侧导航 → 卷

#### 3.3.1 手动创建卷

**操作步骤**：

1. 点击"新建卷"按钮。
2. 填写卷名（如"第一卷：崛起"）、卷描述、目标字数。
3. 点击"确认"创建。

#### 3.3.2 AI自动分卷

**操作步骤**：

1. 点击"AI自动分卷"按钮。
2. AI将根据总纲内容自动分析故事结构，将小说拆分为若干卷。
3. 每卷自动生成卷名、描述和建议字数。
4. 用户可预览分卷方案后批量确认创建。

#### 3.3.3 编辑与删除卷

点击卷卡片上的编辑图标可修改卷信息，点击删除图标可删除卷（需确认）。

### 3.4 章节管理

**操作路径**：项目详情 → 左侧导航 → 卷 → 点击某卷进入章节列表

#### 3.4.1 章节状态说明

每个章节具有以下状态，依次流转：

| 状态 | 说明 |
|------|------|
| 待生成（outline） | 章节已创建，仅有细纲或标题 |
| 已生成（generated） | AI已生成正文初稿 |
| 已润色（polished） | AI已对正文进行文字润色 |
| 已去AI味（de_ai） | AI已去除文本中的AI写作痕迹 |
| 终稿（final） | 作者确认的最终版本 |

#### 3.4.2 新建章节

在章节列表页面，点击"新建章节"按钮，输入章节标题，系统自动创建章节文件。

#### 3.4.3 AI生成章节（四步流水线）

这是软件的核心功能，包含四个连续的AI处理步骤：

**第一步：生成细纲**

1. 在章节编辑页面的AI生成面板中，点击"生成细纲"按钮。
2. AI自动加载当前卷的大纲、人物设定作为上下文。
3. 根据上下文生成详细的章节场景细纲，包含场景划分、关键事件、人物出场等。
4. 生成完成后，细纲显示在编辑区上方，用户可编辑调整。

**第二步：生成正文**

1. 确认细纲后，点击"生成正文"按钮。
2. AI综合细纲、人物设定、世界观、上一章摘要等上下文信息，生成3000-5000字的章节正文。
2. 文本以流式方式实时输出到编辑区，用户可实时看到生成过程。
3. 生成完成后，章节状态自动更新为"已生成"。

**第三步：AI润色**

1. 点击"AI润色"按钮。
2. AI对正文进行深度润色，增强文字表现力、对话自然度、环境描写细腻度。
3. 润色完成后，状态更新为"已润色"。

**第四步：去AI味**

1. 点击"去AI味"按钮。
2. AI对文本进行针对性处理：打散AI常见的重复句式结构、删除机械化连接词、增强口语化表达和真实感。
3. 处理完成后，状态更新为"已去AI味"。

**停止生成**：在AI生成过程中，可随时点击"停止"按钮终止当前生成任务。

#### 3.4.4 章节编辑器

章节编辑器采用分屏布局：

- **左侧编辑区**：支持Markdown格式的文本编辑，提供格式工具栏（加粗、斜体、引用、分割线）。
- **右侧预览区**：实时渲染Markdown预览。
- **字数统计**：编辑区底部实时显示中文字数（CJK字符每个计为1字，英文按空格分词计数）。
- **自动保存**：停止输入2秒后自动保存，也可按Ctrl+S手动保存。
- **流式生成时自动滚动**：AI生成文本时，编辑区自动滚动到最新内容位置。

#### 3.4.5 AI工具箱

在章节编辑页面，还提供以下AI辅助工具：

**生成章节摘要**：

1. 点击"生成摘要"按钮。
2. AI自动阅读当前章节内容，生成精炼的章节摘要。
3. 摘要保存到项目的记忆系统中，后续章节生成时自动作为上下文引用，保障长篇连载的剧情连贯性。

**一致性检查**：

1. 点击"一致性检查"按钮。
2. AI全面扫描当前章节内容，检查以下问题：
   - 人物行为是否符合其性格设定（是否OOC）
   - 是否与已建立的世界观规则冲突
   - 时间线是否合理
   - 是否存在前后矛盾的情节
3. 检查结果以列表形式展示，标注问题位置和修改建议。

**卡文助手**：

1. 当写作遇到瓶颈时，点击"卡文助手"按钮。
2. AI根据当前剧情进展和人物状态，提供多个剧情方向建议。
3. 建议包含可能的冲突点、爽点、伏笔设置等创意参考。

**AI生成标题**：

1. 点击"生成标题"按钮。
2. AI根据章节内容生成5个有吸引力的候选标题供选择。

### 3.5 人物管理

**操作路径**：项目详情 → 左侧导航 → 人物

#### 3.5.1 创建人物

**手动创建**：

1. 在人物列表页面，点击"新建人物"按钮。
2. 填写人物信息：
   - **基本信息**：姓名（必填）、性别、年龄
   - **角色定位**：主角/反派/配角/龙套
   - **性格与背景**：性格特征、个人背景
   - **技能与物品**：技能列表、持有物品
   - **说话风格**：角色的对话特色
   - **目标与秘密**：角色的行动目标、隐藏的秘密
3. 点击"保存"创建人物。

**AI生成人物**：

1. 点击"AI生成人物"按钮。
2. 输入简要的人物描述（如"一个性格孤僻的天才剑客，幼年经历灭门惨案"）。
3. AI自动生成包含完整设定的人物信息（姓名、性格、背景、技能、说话风格等）。
4. 用户可对生成结果进行修改后保存。

#### 3.5.2 人物详情

点击人物卡片进入详情页面，包含四个选项卡：

- **基本信息**：查看和编辑姓名、性别、年龄、角色定位。
- **性格与背景**：查看和编辑性格特征与个人背景故事。
- **技能与物品**：管理角色的技能列表和持有物品列表。
- **关系管理**：管理该角色与其他角色的关系。

#### 3.5.3 关系管理

在人物详情的关系管理选项卡中：

1. 点击"添加关系"按钮。
2. 选择目标人物、关系类型（朋友/敌人/恋人/师徒/上下级/血缘/竞争/同盟/背叛共9种）。
3. 填写关系描述。
4. 设置关系强度（1-10级滑块）。
5. 保存后，关系以带颜色标签的形式展示，不同关系类型使用不同颜色区分。

---

### 3.6 人物关系图谱

**操作路径**：项目详情 → 左侧导航 → 图谱

#### 3.6.1 图谱查看

进入图谱页面后，系统自动加载所有人物及其关系数据，以可视化图谱形式展示：

- **节点**：代表人物，按角色类型着色（主角金色、反派红色、配角蓝色、龙套灰色）。
- **边**：代表关系，按关系类型着色，线条粗细反映关系强度。
- **交互**：支持鼠标拖拽移动节点、滚轮缩放、点击节点查看人物信息。

#### 3.6.2 布局切换

图谱支持三种布局方式：

1. **力导向布局（Force）**：节点通过模拟力的作用自然分布，适合展示复杂关系网络。
2. **环形布局（Circular）**：节点沿圆周均匀排列，适合查看全貌。
3. **层次布局（Dagre）**：节点按层级排列，适合展示有向关系。

在布局下拉框中选择即可切换。

#### 3.6.3 AI自动分析关系

1. 点击"AI分析关系"按钮。
2. AI扫描所有已撰写的章节内容，自动识别人物之间的互动和关系。
3. 分析完成后，自动创建或更新人物关系图谱中的关系边。
4. 用户可在结果列表中逐一确认或删除自动识别的关系。

#### 3.6.4 手动管理关系

- **添加关系**：点击"添加关系"按钮，选择源人物和目标人物，设置关系类型和强度。
- **删除关系**：在关系列表中点击删除图标移除指定关系。

### 3.7 设置管理

**操作路径**：左侧导航 → 设置

#### 3.7.1 AI Provider管理

**添加Provider**：

1. 在设置页面的"AI Provider"区域，点击"添加Provider"按钮。
2. 填写以下信息：
   - 名称：自定义名称，如"OpenAI"、"DeepSeek"等
   - Base URL：AI服务的API地址
   - API Key：访问密钥
   - 模型：模型名称，如"gpt-4o"、"deepseek-chat"等
   - Temperature：控制输出随机性，范围0-2，默认1.0
   - Max Tokens：单次输出最大token数
   - Top P：核采样参数，范围0-1
   - 上下文长度：最大上下文窗口大小
3. 点击"测试连接"验证配置，测试成功即激活。

**管理Provider**：

- **切换激活**：点击Provider卡片上的激活/停用按钮。
- **删除Provider**：点击删除图标，确认后删除。

#### 3.7.2 Prompt模板管理

设置页面展示所有可用的Prompt模板，共14种：

| 模板名称 | 用途 |
|----------|------|
| 大纲润色 | 优化大纲的故事结构和表达 |
| 大纲扩写 | 将简要大纲扩展为详细大纲 |
| 章节细纲生成 | 根据卷纲生成章节场景细纲 |
| 章节正文生成 | 根据细纲生成章节正文 |
| 章节润色 | 增强章节文字表现力 |
| 去AI味重写 | 消除AI写作痕迹 |
| AI人物生成 | 根据描述生成完整人物设定 |
| AI人物关系分析 | 自动分析章节中的人物关系 |
| 章节摘要生成 | 生成章节内容摘要 |
| 卡文助手 | 提供剧情方向建议 |
| 章节标题生成 | 生成候选章节标题 |
| AI辅助创建项目 | 自动生成项目初始内容 |
| AI自动分卷 | 根据总纲自动拆分卷 |
| 一致性检查 | 检查内容前后一致性 |

用户可查看和编辑每个模板的Prompt内容。每个项目可以保存自定义的Prompt模板到项目的`settings/prompt.json`文件中，也可随时恢复为默认模板。

## 4 数据管理

### 4.1 数据存储位置

所有小说数据存储在用户数据目录下的`novels`文件夹中，完整路径为：

```
C:\Users\<用户名>\AppData\Roaming\ai-novel-assistant\novels\
```

### 4.2 项目目录结构

```
novels\
└── <小说名称>\
    ├── project.json          # 项目元数据
    ├── outline\
    │   ├── 总纲.md            # 总纲文档
    │   ├── 世界观.md           # 世界观设定
    │   └── 支线.md            # 支线剧情
    ├── volumes\
    │   └── <卷名>\
    │       ├── volume.json   # 卷元数据
    │       └── chapter_001.md # 章节文件
    ├── characters\
    │   └── <角色名>.json      # 人物设定
    ├── graph\
    │   └── relations.json    # 人物关系数据
    ├── memory\
    │   ├── timeline.json     # 时间线
    │   └── summaries.json    # 章节摘要
    └── settings\
        └── prompt.json       # 自定义Prompt模板
```

### 4.3 数据备份

用户可直接复制`novels`文件夹进行数据备份。如需迁移至其他电脑，复制该文件夹并在新机器上启动软件即可使用。

## 5 常见问题

**Q：AI生成的文本质量不理想怎么办？**
A：可尝试以下方法：（1）调整Temperature参数，降低值可获得更稳定的输出；（2）优化大纲和人物设定的详细程度，为AI提供更充分的上下文；（3）使用"去AI味"功能进一步优化文本；（4）在章节编辑器中手动修改后重新生成。

**Q：章节状态可以回退吗？**
A：可以。在AI生成面板中，可从任意步骤重新开始执行，新生成的内容将覆盖当前版本。

**Q：如何更换AI服务提供商？**
A：在设置页面添加新的Provider并激活即可，原有Provider可保留或删除。

**Q：支持离线使用吗？**
A：大纲编辑、章节编辑、人物管理等核心编辑功能支持离线使用。AI生成、润色等需要调用AI接口的功能需要网络连接。
